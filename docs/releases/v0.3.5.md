# TaipanStack v0.3.5 — Re-release of v0.3.4

## Overview

v0.3.5 is a **re-release** of the intended `v0.3.4` content.

The original `v0.3.4` tag was accidentally published to PyPI in an **incomplete/pre-release state**. Since PyPI does not permit deletion or overwriting of published versions, the corrected release is being published as `v0.3.5`. The code, features, and fixes are identical to what was planned for `v0.3.4`.

> [!NOTE]
> If you installed `taipanstack==0.3.4` from PyPI before this release, you received an incomplete build. Upgrade to `v0.3.5` to get the full intended release.

---

## What's included

All features and fixes from the intended `v0.3.4` release are present:

- 🔒 **Security Fix**: Plaintext password storage in `UserService.create_user` is now corrected — passwords are hashed before storage (PR #93)
- 🔐 **New `password` module**: `hash_password`, `verify_password`, `generate_secure_token`
- 🚀 **Async resilience**: Native `async def` support for `@retry` and `@circuit_breaker`
- 🛡️ **Pydantic security types**: `SafeUrl`, `SafePath`, `SafeCommand`, `SafeProjectName`
- 🕵️ **Structlog data masking**: `mask_sensitive_data_processor` redacts sensitive keys automatically
- 🧪 **Test coverage**: ~35 new/updated tests across validators, result types, logging, filesystem, and password modules
- 🧹 **Code health**: Removed unused imports, defined `__all__`, removed redundant `from __future__ import annotations`
- ⚡ **Performance**: `generate_pre_commit_config` uses list + `join` instead of `+=`

See [v0.3.4 release notes](./v0.3.4.md) for full details of all changes.

---

## Upgrade

```bash
pip install --upgrade taipanstack==0.3.5
```

No breaking changes — upgrade is safe for all v0.3.x installations.
