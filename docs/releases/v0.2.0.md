# TaipanStack v0.2.0 — Project Rename & Modular Architecture

**Date:** February 3, 2026

## Overview

v0.2.0 marks the **rename from "Stack" to "TaipanStack"** and introduces a fully modular architecture with security modules, expanded test coverage, and PyPI distribution.

---

## ⚠️ Breaking Changes

- **Project renamed**: "Stack" → "TaipanStack"
- **All imports changed**: `stack.*` → `taipanstack.*`
- **Bootstrapper renamed**: `stack_bootstrapper.py` → `taipanstack_bootstrapper.py`

---

## What's New

### 🏗️ Architecture
- Modular architecture with dedicated `core/`, `config/`, `security/`, and `utils/` packages
- Result types (`Ok`/`Err`) for explicit error handling
- Python version-aware optimization system (`core.compat`, `core.optimizations`)
- Configuration models with Pydantic v2 (`config.models`, `config.version_config`)

### 🛡️ Security Modules
- `security.guards` — path traversal and command injection protection
- `security.sanitizers` — XSS, filename, SQL, and env value sanitization
- `security.validators` — email, URL, project name, and Python version validation

### 🔧 Utilities
- `utils.retry` — exponential backoff retry decorator
- `utils.circuit_breaker` — circuit breaker pattern
- `utils.metrics` — `@timed` and `@counted` decorators
- `utils.logging` — structured logging with structlog
- `utils.filesystem` — safe file operations

### 📦 Distribution
- PyPI package metadata and URLs
- Package entry points for distribution
- Added `polyfactory` (>=2.0.0) for test data generation
- Added `pydantic-settings` (>=2.0.0) for configuration management

### 🧪 Testing
- 569 tests passing with 89% coverage
- Multi-OS CI matrix (Ubuntu, macOS, Windows)
- Multi-Python version testing (3.11–3.14)
