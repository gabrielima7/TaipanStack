# Release Notes - TaipanStack v0.3.9

## [0.3.9] - 2026-03-11

This release focuses on security robustness, performance optimizations, and test suite stabilization.

### 🛡️ Security
- **Critical Path Traversal Fix**: Added rigorous protections in filesystem utilities (`find_files`, `get_file_hash`) to prevent access outside the base directory. (#160)
- **Sensitive Data Masking**: Implemented automatic masking of secrets (passwords, tokens, keys) in the structured logging system. (#162)
- **New Security Types (Pydantic v2)**: Implemented `SafeHtml` and `SafeSqlIdentifier` natively compatible with FastAPI/Pydantic schemas, protecting against XSS and SQL Injection.
- **JWT Decode Improvement**: The "none" algorithm is now explicitly rejected before processing, mitigating signature bypass attacks. (#169)
- **Guards Optimization**: Unified regex patterns for verifying sensitive environment variables, increasing runtime protection efficiency. (#167)

### 🧩 Core Features (Observability, Resilience, and Cache)
- **Observability Module (`context`)**: Added `correlation_id` tracking via `contextvars` integrating distributed tracing into `structlog` with automatic scope limits.
- **Resilience Decorators (`resilience`)**:
    - `@fallback`: Intercepts exceptions and business errors, returning safe fallback values.
    - `@timeout`: Manages maximum execution time in synchronous and asynchronous functions, preventing eternal locks.
- **Intelligent Cache (`cache`)**: New `@cached(ttl)` decorator that memoizes only successful results (`Ok`), automatically ignoring and recomputing failures (`Err`).

### 🧪 Quality and Testing (QA)
- **Concurrency Tests Stabilization**: Resolved flakiness issues in synchronous and asynchronous execution limit tests using event-based synchronization. (#165)
- **Coverage Increase**:
    - New tests for path sanitization and directory traversal handling. (#168)
    - Tests for purely numeric SQL identifiers. (#166)
    - Edge case coverage for excessively long file extensions. (#163)
    - Unit tests for the main module (`app.main`). (#159)

### ⚡ Performance
- **Hashing Optimization**: Refactored chunk iteration in large files to use `functools.partial`, reducing lambda call overhead. (#164)
- **JWT Performance**: Micro-optimization in the validation of allowed algorithms. (#169)

### 🧹 Chore
- **Refactoring**: Removed syntax redundancies (unnecessary `elif` statements). (#161)
- **Refactoring**: Updated logging infrastructure to use standardized internal utilities. (#162)
- **Linting**: Restored Ruff standard formatting across all module files.
